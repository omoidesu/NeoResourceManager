## 资源分类表 category
```sql
id varchar(255) primary key,
name varchar(255) not null comment '资源分类名称，如游戏、软件、单图、多图、电影、番剧、音声、小说、网站',
emoji varchar(255) comment '分类图标/emoji',
reference_id varchar(255) comment '关联到 dict_data.id 的分类元数据配置',
sort int comment '排序值',
is_deleted tinyint default 0 comment '删除标记'
```

## 主要关系说明

### 1. 分类与资源
- `resource.category_id -> category.id`
- `category.reference_id -> dict_data.id`
  - 用于描述分类元配置，例如扩展表名、作者文案、启动按钮文案、允许扩展名等

### 2. 资源与扩展表
- `game_meta.resource_id -> resource.id`
- `software_meta.resource_id -> resource.id`
- `single_image_meta.resource_id -> resource.id`
- `multi_image_meta.resource_id -> resource.id`
- `video_meta.resource_id -> resource.id`
- `asmr_meta.resource_id -> resource.id`
- `novel_meta.resource_id -> resource.id`
- `website_meta.resource_id -> resource.id`

### 3. 资源与统计/日志
- `resource_stat.resource_id -> resource.id`
- `resource_log.resource_id -> resource.id`

### 4. 资源与人物/站点/标签/分类项
- `author_work.author_id -> author.id`
- `author_work.resource_id -> resource.id`
- `store_work.resource_id -> resource.id`
- `store_work.store_id -> dict_data.id`
- `actor.resource_id -> resource.id`
- `tag_resource.resource_id -> resource.id`
- `tag_resource.tag_id -> tag.id`
- `type_resource.resource_id -> resource.id`
- `type_resource.type_id -> resource_type.id`

### 5. 资源设计特点
- `resource` 负责所有通用字段
- 具体分类差异通过 `dict_data.extra.extendTable` 决定落到哪张 meta 表
- 贩售网站、标签、分类项、作者均为多对多/一对多关系
- 运行状态通过 `resource_log.end_time is null` 判断
- 收藏、通关、丢失状态直接存放在 `resource` 主表
