# 更新记录

本文档用于记录项目各版本的主要变更。

## 0.2.0 - 2026-04-06

### 新增
- 新增 Pixiv 图片信息插件，支持根据作品 ID 获取标题、画师、标签、作品类型等信息。
- 新增综合漫画信息插件，支持按多个漫画来源顺序搜索并回填漫画信息。
- 新增漫画专用阅读器，支持连续阅读与单页阅读两种模式。
- 新增漫画阅读进度记录，并将进度持久化到数据库中。
- 新增漫画详情中的阅读进度展示、图片预览区和整目录图片浏览能力。
- 新增漫画元数据字段“汉化者”，支持手动编辑和插件自动回填。
- 新增漫画批量导入能力，可按目录批量添加漫画并可选自动抓取漫画信息。
- 新增资源卡片右键菜单中的危险删除操作，统一支持删除资源和删除本地文件。
- 新增 E-Hentai 中文标签翻译接入，使用 EhTagTranslation 数据库对标签做中文化。

### 调整
- 将漫画信息插件按网站源拆分为独立模块，聚合入口负责统一调度。
- 调整 E-Hentai 候选匹配策略，改为基于搜索结果与元数据的本地评分，而不是直接使用默认排序。
- 调整漫画默认搜索词生成规则，清除目录名中各种括号内内容后再用于搜索。
- 调整漫画批量导入交互，不再要求手动填写贩售网站与作品 ID，改为直接使用清洗后的搜索词自动获取信息。
- 调整图片与漫画详情中的统计文案，将“启动”统一替换为“浏览”。
- 调整漫画详情展示逻辑，隐藏贩售网站与 `storeWork` 记录，重点展示阅读相关信息。
- 调整漫画阅读交互，补充缩放、拖动、点击翻页、双击切换缩放、骨架屏等细节体验。
- 调整新增图片时的默认名称逻辑，默认去掉文件扩展名。

### 修复
- 修复图片站点插件无法匹配导致 Pixiv 获取信息报“未找到可用插件”的问题。
- 修复单图获取 Pixiv 信息时使用错误站点 ID 的问题。
- 修复漫画详情中内容溢出卡片边框、布局拥挤和按钮区域挤压的问题。
- 修复漫画封面未变更时重复生成缩略图导致写入失败的问题。
- 修复漫画详情在资源丢失时错误显示日志的问题。
- 修复漫画阅读中连续阅读定位偏移的问题，跳转到保存页码或指定页时会先加载前序页面后再定位。
- 修复漫画目录图片在部分路径场景下预览不显示的问题。

## 0.1.1 - 2026-04-01

### 新增
- 新增通过 MTool 启动游戏的能力，接入设置项、右键菜单、引擎支持判断和 Hook 映射。
- 新增通过 Locale Emulator 启动游戏的能力，接入设置项和启动流程。
- 新增更细化的游戏引擎识别逻辑，补充了 Kirikiri、Wolf RPG、RPG Maker 多分支、Ren'Py、SMILE GAME BUILDER、Bakin、Pixel Game Maker MV 等识别。
- 新增 MTool 可用引擎标记与对应 Hook 信息，方便后续继续扩展 MTool 支持。
- 新增在检测到游戏引擎后自动回写数据库的能力，前提是该游戏原本未配置引擎。
- 新增多选模式下的批量添加标签、批量移除标签、批量添加分类、批量移除分类。
- 新增 SMILE GAME BUILDER 图标资源及对应映射。
- 新增版本更新记录文件。

### 调整
- 细化了 `seed` 中的游戏引擎列表，使其更接近 MTool 的引擎粒度。
- 优化了 SMILE GAME BUILDER 的识别规则，兼容旧版打包结构。
- 优化多选工具栏在窄窗口下的布局，避免操作按钮被资源卡片遮挡。
- 将“退出多选”和“批量删除”拆分为不同交互，避免误触删除确认。
- 调整批量标签/分类操作的提示文案，改为基于实际受影响资源数量显示。
- 恢复在未配置 LE 路径时禁用“转区启动”菜单项。
- 调整 Everything 提示逻辑，在未启用 Everything 功能时不再提示缺失。
- 补充了设置页和添加游戏表单中的部分 placeholder 文案。

### 修复
- 修复部分 SMILE GAME BUILDER 游戏无法正确识别的问题。
- 修复多选状态下点击“退出多选”会弹出批量删除确认框的问题。
- 修复批量移除标签时提示数量与实际受影响资源不一致的问题。
